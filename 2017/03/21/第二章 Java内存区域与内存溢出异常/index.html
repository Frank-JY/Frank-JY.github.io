第二章 Java内存区域与内存溢出异常

标签（空格分隔）： Java虚拟机

##2.2

  • 运行时数据区域：
    1、程序计数器：
    线程私有，不会抛OutOfMemoryError
    2、Java虚拟机栈：
    线程私有，每个方法执行时都会创建一个栈帧用于存储局部变量表、操作数栈、动态链接、方法出口等信息。局部变量表所需的内存空间在编译期间完成分配，当进入一个方法时，这个方法需要在帧中分配多大的局部变量空间完全确定的，在方法运行期间不会改变局部变量表的大小。
    3、本地方法栈：
    虚拟机栈为执行Java方法（也就是字节码）服务，而本地方法栈则为虚拟机使用到的Native方法服务。
    4、Java堆：
    是java虚拟机所管理的内存中最大的一块。Java堆是垃圾收集器管理的主要区域，也被称为“GC堆”。Java堆中还细分为：新生代和老年代;再细致一点有Eden空间、From Survivor空间、To Suvivor空间等。
    5、方法区：
    用于存储已被虚拟机加载的类信息、常量、静态常量、即时编译器编译后的代码等数据。很多人愿意把方法区称为“永久代”。在目前的JDK1.7的HotSpot中，已经把原本放在永久代的字符串常量池。这区域的内存回收目标主要是针对常量池的回收和对类型的卸载。

##2.3对象的探秘

  • 对象的创建：虚拟机遇到一条nuw指令时，首先将去检查这个指令的参数是否能在常量池中定位到一个符号引用，并且检查这个符号引用代表的类是否已被加载、解析和初始化过。如果没有，那必须先执行相应的类加载过程。在类加载完成后，接下来虚拟机将为新生对象分配内存。
  • 内存分配方式：
    1、指针碰撞。
    2、空闲列表。
  • 对象的内存布局：
    对象在内存中存储的布局可以分为3块区域：
    1、对象头（Header）：第一部分用于存储对象自身的运行时数据。另一部分是类型指针。
    2、实例数据（Instance Data)：存储对象真正存储的有效信息。
    3、对齐填充（Padding）：仅仅起着占位符的作用。
  • 对象的访问定位：
    Java程序需要通过栈上的reference数据来操作堆上的具体对象。目前主流的访问方式有使用句柄和直接指针两种。
